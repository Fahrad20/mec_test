# Mecenate Feed App

Мобильное приложение для платформы Mecenate — лента постов с бесконечной подгрузкой, детальный просмотр поста с комментариями, лайками и WebSocket-обновлениями в реальном времени.

## Функционал

- ✅ **Лента постов**: Cursor-пагинация через React Query (`useInfiniteQuery`)
- ✅ **Фильтрация по табам**: Все / Бесплатные / Платные (серверная фильтрация по `tier`)
- ✅ **Детальный просмотр поста**: Hero-изображение, полный текст, автор
- ✅ **Лайки**: Оптимистичное обновление через MobX, анимация Reanimated 2, тактильная отдача (Haptics)
- ✅ **Комментарии**: Ленивая подгрузка через `useInfiniteQuery`, отправка с оптимистичным добавлением
- ✅ **WebSocket**: Обновления лайков и комментариев в реальном времени, автопереподключение
- ✅ **Темная и светлая темы**: Адаптивные дизайн-токены
- ✅ **Локализация**: Русский и английский языки (i18next)
- ✅ **Архитектура**: Строгое разделение компонентов (Component-Driven, Feature-Sliced паттерны), где стили (`styles.ts`), типы (`types.ts`) и логика (`index.tsx`) лежат раздельно.

## Архитектурные решения

### MobX vs React Query

Проект использует **две системы управления состоянием**, каждая для своих целей:

- **React Query** — управление серверным состоянием (кэширование, пагинация, рефетч). Используется для лент постов (`useFeed`), деталей поста (`usePostDetail`), и комментариев (`useComments`, `useSendComment`).

- **MobX** — управление доменным состоянием на клиенте. `postStore` хранит состояние лайков с оптимистичными обновлениями и rollback'ом при ошибке. `realtimeStore` управляет WebSocket-соединением и маршрутизирует входящие события. `themeStore` управляет темой.

Такое разделение позволяет:
- React Query отвечать за синхронизацию с сервером и кэширование
- MobX мгновенно обновлять UI без ожидания ответа сервера (оптимистичные обновления)
- WebSocket-события через MobX обновлять данные в React Query cache без полного рефетча

## Переменные окружения

| Переменная | Описание | Пример |
|---|---|---|
| `EXPO_PUBLIC_API_BASE_URL` | Базовый URL API | `https://k8s.mectest.ru/test-app` |
| `EXPO_PUBLIC_WS_URL` | URL WebSocket-сервера | `wss://k8s.mectest.ru/test-app/ws` |

## Установка

```bash

cd <repo>
npm install
cp .env.example .env
# Заполните значения в .env при необходимости
npx expo start
```

## Запуск на устройстве

1. Установите приложение **Expo Go** из App Store
2. Запустите `npx expo start`
3. Отсканируйте QR-код камерой iPhone

1. Установите приложение **Expo Go** из Google Play
2. Запустите `npx expo start`
3. Отсканируйте QR-код в приложении Expo Go

## Известные ограничения

- **In-memory backend**: Данные на сервере хранятся в памяти и сбрасываются при перезапуске.
- **Без регистрации**: Аутентификация через любой UUID, без реальной системы пользователей.
- **WebSocket broadcast**: Все клиенты получают все события — фильтрация по postId происходит на клиенте.
- **Платный контент**: Для paid-постов `body` возвращается пустым — реальная система оплаты не реализована.
